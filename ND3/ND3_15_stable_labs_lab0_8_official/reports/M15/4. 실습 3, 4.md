# 실습 3: 전역 경로 알고리즘 비교 보고서

본 실습은 로봇이 출발지에서 목적지까지 주행할 때 전역 지도 데이터를 기반으로 최적의 궤적을 연산하는 다양한 '전역 경로 계획 알고리즘(Global Path Planner)'의 성능을 계획 시간, 경로 길이, 탐색 노드 수 측면에서 수치적으로 비교·분석하는 실습입니다.

---

## 1. 실습 개요
* **목적**: 동일한 지도 환경 내에서 각 전역 경로 알고리즘이 가지는 수학적 탐색 특성과 연산 효율성을 교차 검증합니다.
* **대상 알고리즘**: Dijkstra, A*, Weighted A*, Greedy, RRT

---

## 2. 실습 실행 방법 및 환경

모든 테스트는 가제보 물리 엔진의 실시간 구동 없이, 사전 저장된 점유 격자 지도(`lab_map.yaml`) 데이터를 분석 프로그램 인자로 주입하여 독립적인 연산 성능을 측정하는 방식으로 진행되었습니다.

```bash
cd ~/ND3_15_stable_labs_lab0_8_official
source /opt/ros/humble/setup.bash
source scripts/env.sh

# 1. 장애물이 적은 구역 테스트 (Test 1)
python3 scripts/lab4_practice.py --map maps/lab_map.yaml --start-x -0.95 --start-y -1.5 --goal-x 0.8 --goal-y 1.6 --algorithms dijkstra astar weighted_astar greedy rrt

# 2. 복잡한 우회로 구역 테스트 (Test 2)
python3 scripts/lab4_practice.py --map maps/lab_map.yaml --start-x -0.9 --start-y -1.5 --goal-x 1.1 --goal-y 1.8 --algorithms dijkstra astar weighted_astar greedy rrt

# 3. 오프라인 종합 벤치마크 테스트
bash scripts/lab4_planner_offline_md.sh
````

## 4. 데이터 기반 알고리즘별 논리적 비교 분석

실측 데이터 수치를 바탕으로 각 알고리즘의 동작 특성을 논리적으로 교차 진단할 수 있습니다.

- **Dijkstra (최적 경로 보장 / 대량 탐색형)**
    
    - 실험 1과 2 모두에서 A*와 완벽히 동일한 최단 경로 길이(3.825m / 4.128m)를 찾아내어 수학적 최적성을 입증했습니다.
        
    - 그러나 휴리스틱(거리 힌트) 지표가 없기 때문에 격자 지도의 모든 방향을 탐색하여 확장 노드 수(최대 7,467개)가 가장 많았고, 이로 인해 계획 시간도 70ms 이상으로 가장 길게 소요되었습니다.
        
- **A* (최적 경로 보장 / 연산 효율형)**
    
    - Dijkstra와 소수점 셋째 자리까지 일치하는 최단 거리 경로를 도출하면서도, 거리 휴리스틱 힌트를 활용함으로써 확장 노드 수를 약 6분의 1 수준(1,141개 ~ 1,268개)으로 크게 줄였습니다. 계획 시간 역시 11ms 대로 유지되어 효율성이 검증되었습니다.
        
- **Weighted A* 및 Greedy (초고속 연산 / 준최적형)**
    
    - 두 알고리즘 모두 목적지 방향 탐색 강도를 높여 확장 노드 수를 격자 경로 셀 개수와 유사한 수준(63~73개)으로 극소화했습니다.
        
    - 계획 속도는 1ms 이하로 모든 알고리즘 중 가장 빠릅니다. 다만, 종합 벤치마크 결과에서 확인할 수 있듯이 복잡한 환경에서는 최적 경로(2.309m)보다 소폭 우회하는 경로(Greedy: 2.338m)를 도출하여 속도와 정확도 간의 트레이드오프(Trade-off) 관계를 보여줍니다.
        
- **RRT (무작위 샘플링 기반 / 비최적형)**
    
    - 임의의 난수 좌표를 샘플링하여 트리 구조를 확장하므로 최단 경로 탐색에는 부적합합니다[cite: 1]. 경로 길이가 가장 길고(최대 5.503m)[cite: 1], 환경 변화에 따라 연산 시간이 급증하는 변동성(최대 153.749ms)을 나타냈습니다[cite: 1].
        

## 5. 실습 결론

- 자율주행 로봇의 전역 경로 설계 시, 연산 자원이 유한한 임베디드 환경에서는 Dijkstra보다 탐색 영역이 효율적인 A* 계열이나 제어 주기를 단축할 수 있는 Weighted A* 알고리즘 프로필을 적용하는 것이 시스템 안정성 측면에서 논리적으로 타당함을 실측 데이터를 통해 증명하였습니다[cite: 1].

# 실습 4: 지역 경로 알고리즘 비교 보고서

본 실습은 로봇이 전역 경로를 따라 주행할 때 실시간 동적 제어 명령(/cmd_vel)을 계산하는 '지역 제어기(Local Controller - DWB)'의 두 가지 제어 정책인 Safe(안전 주행)와 Agile(민첩 주행)의 주행 효율성 및 정밀도를 수치적으로 비교·분석하는 실습입니다.

---

## 1. 실습 개요
* **목적**: 속도 제한 및 제어 민감도 설정에 따른 로봇의 목표점 도달 시간, 누적 제어 노력(Effort), 제어 주기당 연산 샘플 수, 최종 위치 오차의 상관관계를 교차 진단합니다.
* **대상 정책**: Safe Policy(안전 우선 제어), Agile Policy(속도 우선 제어)

---

## 2. 실습 실행 방법 및 환경

모든 시뮬레이션은 가제보 물리 엔진의 실시간 구동 없이, DWB 플래너의 속도 제어 파라미터 프로필과 주행 모델을 수학적으로 가동하여 독립적인 오프라인 제어 궤적 데이터를 측정하는 방식으로 수행되었습니다.

```bash
cd ~/ND3_15_stable_labs_lab0_8_official
source /opt/ros/humble/setup.bash
source scripts/env.sh

# 1. 기본 파라미터 기반 제어 비교 테스트 (실험 1)
python3 scripts/lab5_practice.py

# 2. 파라미터 변경 및 목표 거리 확장 테스트 (실험 2)
python3 scripts/lab5_practice.py --safe-v 0.12 --safe-w 0.7 --agile-v 0.30 --agile-w 1.4 --target-x 2.0 --target-y 0.8

# 3. DWB Safe/Agile 오프라인 일괄 벤치마크 및 시각화
bash scripts/lab5_dwb_safe_agile_offline_md.sh
````

- **설정**: Safe (최대 선속도 0.12 m/s, 각속도 0.7 rad/s) / Agile (최대 선속도 0.30 m/s, 각속도 1.4 rad/s)

## 4. 데이터 기반 제어 정책별 논리적 비교 분석

실측 데이터 수치를 바탕으로 두 제어 정책의 차이점을 다음과 같이 논리적으로 설명할 수 있습니다.

- **주행 시간 및 연산 속도 효율성**
    
    - 실험 1의 기본 설정에서 Agile 정책(9.6초)은 Safe 정책(15.0초)에 비해 **목표점 도달 시간을 5.4초 단축**하였으며, 이는 전체 이동 주행 시간을 **36% 감소**시킨 결과입니다.
        
    - 실험 2와 같이 목표 거리를 늘리고 속도 제한을 더 완화했을 때는 주행 속도 격차가 18.4초 대 8.7초로 증가하여, Agile 정책이 Safe 정책보다 **52.7% 빠르게 이동**하는 고효율을 보였습니다.
        
- **제어 반복 주기와 샘플 수(Samples)**
    
    - Safe 정책은 속도가 느린 만큼 목표 지점에 도달할 때까지 더 많은 제어 루프를 돌기 때문에 연산 샘플 수(최대 185개)가 대폭 증가합니다.
        
    - 반면 Agile 정책은 고속으로 구간을 돌파하므로 샘플 수가 최소 88개 수준으로 낮아져, 제어 연산 횟수 자체를 줄이는 이점이 있습니다.
        
- **제어 노력(cmd_effort)과 위치 정밀도(final_error_m)의 트레이드오프**
    
    - Agile 정책은 급격한 가속과 빠른 조향 제어를 수행하므로 주행 과정에서 모터에 가해지는 제어 노력 수치(`cmd_effort`: 25.272 ~ 25.340)가 Safe 정책보다 항상 높게 나타납니다. 이는 고속 주행 시 로봇의 기계적 진동이나 차체 흔들림 가능성이 증가할 수 있음을 나타냅니다.
        
    - 최종 위치 정밀도의 경우, 두 정책 모두 목표 오차 반경을 3.7cm ~ 3.9cm 사이로 안정적으로 제어하여 목적지 수렴성 측면에서는 모두 합격점을 받았습니다[cite: 1].
        

## 5. 실습 결론

- 본 실습 데이터를 통해 DWB 플래너 운영 시 고정된 주행 설정을 고집하기보다, 상황에 맞는 정책 적용이 필수적임을 도출했습니다[cite: 1].
    
- 보행자가 많거나 정밀한 충돌 회피가 필요한 복잡하고 좁은 환경에서는 모터 부하가 적고 안정적인 **Safe 프로필**이 타당합니다[cite: 1]. 반면, 장애물이 없는 개방된 공간이나 물류 이송 등 신속성이 최우선인 산업 환경에서는 이동 시간을 절반 이상 절약할 수 있는 **Agile 프로필**을 선택하는 것이 논리적으로 최적의 제어 전략임을 실측 수치로 입증하였습니다[cite: 1].
