# 실습 5: 장애물 삽입 경로 주행 보고서

본 실습은 로봇의 주행 경로 상에 가상의 장애물(붉은 상자)을 동적으로 삽입하여, 내비게이션 시스템(Nav2)의 코스트맵(Costmap)이 이를 실시간으로 반영하고 장애물을 회피하여 안전한 우회 경로를 재생성(Replanning)하는 과정을 검증하는 실습입니다.

## 1. 실습 개요

- **목적**: 동적 장애물 배치 시 로봇의 장애물 탐지 성능 및 실시간 우회 주행 알고리즘의 유효성을 검증합니다.
    
- **핵심 기능**: 레이저 스캔 데이터(`/scan`)를 수신한 코스트맵 서버가 정적 지도에 실시간 충돌 위험 영역을 중첩 표시하고, 로컬 플래너가 해당 영역을 회피하는 제어 명령(`/cmd_vel`)을 실시간으로 도출합니다.
    

## 2. 실습 환경 및 실행 방법

모든 터미널 창은 아래의 공통 환경 변수 소싱 및 디렉토리 이동 명령을 선행한 후 실습 명령어를 실행합니다.

### [터미널 1] 물리 시뮬레이터(Gazebo) 가동

가상 로봇이 동작하는 3D 물리 환경을 실행합니다.

Bash

```
cd ~/ND3_15_stable_labs_lab0_8_official
source /opt/ros/humble/setup.bash
source scripts/env.sh
python3 scripts/lab1_practice.py --action start-gazebo
```

### [터미널 2] 내비게이션(Nav2) 서버 실행 및 초기 위치 지정

저장한 2D 격자 지도를 불러오고 시각화 도구(RViz2)를 구동합니다.

Bash

```
cd ~/ND3_15_stable_labs_lab0_8_official
source /opt/ros/humble/setup.bash
source scripts/env.sh
bash scripts/lab3_start_nav2_official.sh maps/lab_map.yaml
```

- **필수 조작**: RViz2 창이 켜지면 상단 툴바의 **`2D Pose Estimate`** 버튼을 클릭하여 지도상 로봇의 초기 위치와 바라보는 방향을 마우스로 지정해 줍니다.
    

### [터미널 3] 동적 장애물 투하 및 자율주행 명령

#### 1. 가상 장애물 상자(`lab_box`) 투하

로봇의 정면 부근($x=0.5, y=0.0$) 좌표에 장애물을 생성합니다.

Bash

```
cd ~/ND3_15_stable_labs_lab0_8_official
source /opt/ros/humble/setup.bash
source scripts/env.sh
python3 scripts/lab6_practice.py --action spawn --x 0.5 --y 0.0 --z 0.25 --model lab_box
```

#### 2. 우회 주행을 위한 목적지(Goal) 설정

장애물 뒤편의 목적지 좌표를 송신하여 자율주행을 개시합니다.

Bash

```
cd ~/ND3_15_stable_labs_lab0_8_official
source /opt/ros/humble/setup.bash
source scripts/env.sh
bash scripts/lab3_send_goal.sh 1.05 1.50 0.0
```

(추가 복귀 경로 테스트가 필요한 경우 다음 목적지를 인가합니다.)

Bash

```
cd ~/ND3_15_stable_labs_lab0_8_official
source /opt/ros/humble/setup.bash
source scripts/env.sh
bash scripts/lab3_send_goal.sh 0.75 0.25 0.0
```

#### 3. 실습 후 환경 원상복구 (장애물 제거 및 코스트맵 클리어)

설치했던 장애물 오브젝트를 완전히 삭제하고 잔상을 청소합니다.

Bash

```
cd ~/ND3_15_stable_labs_lab0_8_official
source /opt/ros/humble/setup.bash
source scripts/env.sh
python3 scripts/lab6_practice.py --action delete --model lab_box

# 잔상이 남은 경우 코스트맵 리셋 실행
python3 scripts/lab6_practice.py --action clear
```

## 3. 실습 주요 결과 및 분석

### ① 실시간 코스트맵(Costmap) 업데이트 관찰

- **가제보 시뮬레이터**: 명령에 의해 로봇 전방에 빨간색 상자(`lab_box`)가 물리적으로 정상 생성되는 것을 확인했습니다.
    
- **RViz2 시각화**: 로봇의 센서 빔이 상자를 감지하는 즉시, 2D 격자 지도 위에 붉은색과 노란색의 레이어가 덮이면서 해당 공간이 주행 불가능 구역(Lethal Obstacle) 및 충돌 완충 구역(Inflation Layer)으로 실시간 표시되었습니다.
    

### ② 동적 경로 재계획(Dynamic Replanning) 확인

- **경로 계획 결과**: 로봇이 직진 구도를 그리다 장애물을 감지하자마자 기존의 전역 경로선(Global Path)을 폐기하고, 상자의 옆면 경계를 안전하게 우회하여 지나가는 새로운 곡선 경로를 순간적으로 생성하는 것을 확인했습니다.
    
- **주행 피드백**: 모터 제어 명령을 담당하는 제어 서버(`controller_server`)는 목적지 도달까지 실시간 제어 출력을 연속적으로 조절했으며, 충돌 없이 안전하게 목적지에 도달하여 최종적으로 `Goal succeeded` 신호를 출력하였습니다.
    

## 4. 실습 결론

- 정적 지도를 바탕으로 주행 경로를 미리 계획했더라도, 실제 주행 중에 발생하는 실시간 돌발 상황이나 장애물 요소를 감지할 수 있는 '동적 코스트맵'의 유무가 로봇 자율주행의 안전성을 보장하는 핵심 인프라임을 명확하게 검증하였습니다.
