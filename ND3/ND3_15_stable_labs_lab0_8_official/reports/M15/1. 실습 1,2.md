# ROS2 자율주행 실습 보고서: SLAM 및 Navigation

본 보고서는 Ubuntu(WSL2) 환경의 ROS2 Humble 시스템을 활용하여 가상 로봇의 2D 격자 지도를 작성(SLAM)하고, 이를 기반으로 목적지까지 자율주행(Navigation)을 수행한 실습 기록입니다.

---

## 0. 공통 사전 환경 설정
모든 새 터미널 창을 실행할 때마다 ROS2 Humble 환경 및 실습 프로젝트 환경 변수를 동기화하기 위해 아래 명령어를 선행 실행했습니다.
```bash
cd ~/ND3_15_stable_labs_lab0_8_official
source /opt/ros/humble/setup.bash
source scripts/env.sh
````

## 1. 런타임 스모크 체크 (Runtime Smoke Check)

주행 알고리즘 작동 전, 시뮬레이터와 ROS2 간의 기초 데이터 채널 정상 개설 여부를 교차 검증했습니다.

- **실행 명령어**:
    
    Bash
    
    ```
    python3 scripts/lab1_practice.py --action check --timeout 5
    ```
    
- **검증 결과**: `/clock`, `/scan`, `/odom`, `/tf` 토픽의 샘플 데이터 수신이 모두 무결하게 확인되어 센서 인프라 작동 합격 판정(Runtime smoke check passed)을 받았습니다.
    

## 2. 실습 1: SLAM을 활용한 2D 지도 작성 (Mapping)

로봇의 센서 데이터와 바퀴 회전 오도메트리를 수학적으로 대조(스캔 매칭)하여 미지의 공간에 대한 2D 지도를 생성했습니다.

### 구동 프로토콜 (다중 터미널 운영)

1. **터미널 1 (시뮬레이터)**: 가제보 물리 가상 환경 실행
    
    Bash
    
    ```
    python3 scripts/lab1_practice.py --action start-gazebo
    ```
    
2. **터미널 2 (SLAM 연산)**: 실시간 지도 생성 노드 활성화
    
    Bash
    
    ```
    bash scripts/lab2_start_slam_toolbox.sh
    ```
    
3. **터미널 3 (시각화)**: 로봇 스캔 데이터를 그래픽 UI(RViz2)로 모니터링
    
    Bash
    
    ```
    bash scripts/lab2_start_rviz_safe.sh
    ```
    
4. **터미널 4 (수동 조종)**: 키보드를 통한 로봇 주행
    
    Bash
    
    ```
    bash scripts/lab2_teleop.sh
    ```
    

### 지도 주행 및 품질 관리 기준

- 스캔 매칭 왜곡을 방지하기 위해 직선 구간은 서행하고, 회전 구간에서는 로봇을 멈춘 후 부드럽게 제자리 회전시켰습니다.
    
- 누적 오차를 전역적으로 보정(Loop Closure)하기 위해 동일 구간을 최소 2회 이상 중첩 주행했습니다.
    

### 지도 저장 및 최종 검증

- **터미널 5 (지도 백업)**: 지도를 디스크에 영구 기록
    
    Bash
    
    ```
    bash scripts/lab2_save_map.sh maps/lab_map
    ```
    
- **무결성 진단**: `inspect-map` 및 `list-maps` 기능을 통한 분석
    
    - 격자 해상도: `0.05m` (셀당 5cm 수준 정밀도)
        
    - 맵 규격: 가로 111칸 $\times$ 세로 103칸 (약 $5.55\text{m} \times 5.15\text{m}$)
        
    - 가용 이동 공간(`free_cells`): 전체 구역의 `93.7%`
        

## 3. 실습 2: Navigation을 활용한 자율주행

작성된 정적 지도를 바탕으로 목적지 좌표까지 실시간 장애물 회피 주행을 구현했습니다.

### 구동 프로토콜

1. **터미널 1 (시뮬레이터)**: 가제보 실행
    
2. **터미널 2 (내비게이션 서버)**: 저장된 맵 주입 및 Nav2 구동
    
    Bash
    
    ```
    bash scripts/lab3_start_nav2_official.sh maps/lab_map.yaml
    ```
    
3. **초기 위치 설정 (필수 연산)**: RViz2 UI 상단의 `2D Pose Estimate`를 클릭하여 가제보 속 실물 로봇의 좌표와 일치하도록 화살표 드래그를 수행했습니다. 이를 통해 위치 추정 노드(`amcl`)가 수렴하고 안전장치가 해제되었습니다.
    
4. **터미널 3 (목적지 명령 전송)**: 목표 좌표 발송 (X=0.5m 전방 이동 명령)
    
    Bash
    
    ```
    bash scripts/lab3_send_goal.sh 0.5 0.0 0.0
    ```
    

### 2D 평면 좌표계 분석 규칙 (REP 103 기본)

- **X축 (상하축)**: 로봇의 정면 방향 (`+` 전진, `-` 후진)
    
- **Y축 (좌우축)**: 로봇의 측면 방향 (`+` 좌측, `-` 우측)
    

## 4. 트러블슈팅 및 핵심 분석 기록

### ① 노드 타임아웃 및 `!rclpy.ok()` 해결 (좀비 프로세스 정리)

- **문제 상황**: 일부 라이프사이클 노드가 `[TIMEOUT/MISSING]` 대기 상태에 빠지거나 `map_server`가 `!rclpy.ok()`를 뱉으며 다운되는 현상 발생.
    
- **원인**: 이전 실행 시 비정상 종료된 통신 노드 및 그래픽 패널 찌꺼기(좀비 프로세스)들이 백그라운드 메모리에 남아 DDS 네트워크 포트를 차단함.
    
- **해결 조치**: 모든 ROS 통신 노드와 데몬 프로세스를 일괄 사살한 후 네트워크 스택을 깨끗이 초기화하여 복구 성공.
    
    Bash
    
    ```
    pkill -f ros && pkill -f rviz && pkill -f gazebo
    ros2 daemon stop && ros2 daemon start
    ```
    

### ② 이전 목적지 취소 현상 (Goal Preemption)

- **발견 사실**: 주행 도중 다른 목적지를 새로 인가하면 이전 목적지를 완수하지 않고 즉시 새 경로로 선회함.
    
- **이유**: 안전과 즉각적인 지휘 통제를 보장하기 위해, Nav2 액션 서버는 항상 가장 마지막에 들어온 명령을 최우선으로 간주하여 기존 작업을 강제 대체(선점)하도록 설계되어 있음.
    

### ③ 다중 포인트 주행 (Waypoint Following) 방법

- 여러 목적지를 순서대로 자동 완주하게 하려면 RViz2의 모드를 변경해야 합니다.
    
    1. RViz2 내 `Navigation 2 패널`에서 실행 모드를 `Single Goal`에서 `Waypoint` 모드로 변환.
        
    2. 상단 `Nav2 Goal` 버튼을 반복 클릭하여 지도 위에 방문 순서(1번, 2번 등)를 다중 지정.
        
    3. 패널 내 `Start Waypoint Following` 버튼을 클릭하여 시나리오 기반 순차 주행을 시작.
        

## 5. 핵심 기술 용어 및 연산 그래프 요약

- **노드 (Node)**: 독립된 하나의 실행 프로그램 단위 (예: 센서 구동, 경로 계획기 등).
    
- **간선 (Edge)**: 노드를 유기적으로 잇는 통신 화살표. 데이터 주입 구조에 따라 토픽(일방향), 서비스(동기 응답), 액션(비동기 피드백)으로 구분.
    
- **Global Planner**: 정적 지도를 기반으로 목적지까지의 거시적인 최적 경로선(`Global path`)을 설계하는 노드.
    
- **Local Controller**: 동적 장애물을 실시간으로 감지하여 모터 속도 명령(`/cmd_vel`)을 초당 수십 번 연산·출력하는 제어 노드.
