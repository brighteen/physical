# 실습 6: Rosbag 기록 및 확인 보고서

본 실습은 ROS2 내비게이션(Nav2) 주행 중 발생하는 다양한 센서 데이터, 제어 명령, 좌표 변환 정보를 시간선에 따라 파일로 기록(Record)하고, 저장된 메타데이터를 진단하며, 이를 가상 환경 없이 시각화 도구 상에 재현(Play)하는 데이터 로깅 실습입니다.

---

## 1. 실습 개요
* **목적**: 자율주행 중 연산 그래프 내에서 실시간으로 교환되는 핵심 토픽 데이터를 단일 파일로 보존하고 분석하는 법을 익힙니다.
* **핵심 기능**: 로봇의 센서 스트림과 제어 알고리즘 연산 결과를 파일로 저장하여, 실제 로봇이나 시뮬레이터가 작동하지 않는 오프라인 상태에서도 동일한 주행 상황을 완벽하게 재현할 수 있는 개발 환경을 구축합니다.

---

## 2. 실습 실행 방법

### Step 1: 내비게이션 구동 및 목적지 설정
1. 가제보 시뮬레이터와 내비게이션(Nav2) 서버를 실행합니다.
2. RViz2 창에서 `2D Pose Estimate` 버튼을 사용해 로봇의 초기 위치를 지정하여 위치 추정 노드(`amcl`)를 활성화합니다.

### Step 2: rosbag 기록 (Record)
새 터미널을 열고 다음 자동화 스크립트를 실행하여 데이터 녹화를 시작합니다. (기록 결과물은 `results/` 디렉토리 아래에 폴더 형태로 자동 저장됩니다.)
```bash
bash scripts/lab7_record_bag.sh
````

*스크립트 실행 직후, RViz2 창에서 `Nav2 Goal`을 설정하여 로봇이 스스로 목적지까지 주행하게 만듭니다. 주행이 끝나면 녹화 터미널에서 `Ctrl + C`*_를 눌러 기록을 안전하게 종료합니다._

### Step 3: 기록된 데이터 구조 진단 (Info)

녹화가 완료된 백(bag) 파일의 메타데이터 정보와 내부 토픽 수집 내역을 분석하기 위해 아래 명령어를 실행합니다.

Bash

```
ros2 bag info results/nav2_20260714_153826
```

### Step 4: 데이터 재현 및 재생 (Play)

실시간 데이터와의 충돌을 막기 위해 가제보 시뮬레이터와 내비게이션 노드를 완전히 종료한 상태에서, 시각화 도구(RViz2)만 켜고 저장된 데이터를 재생합니다.

Bash

```
# 가제보 없이 RViz2만 단독 실행
bash scripts/lab2_start_rviz_safe.sh

# 다른 터미널에서 백 파일 데이터 재생
ros2 bag play results/nav2_20260714_153826
```

## 3. 실습 주요 결과 분석 (`ros2 bag info` 수치 기반)

실행 결과, 데이터베이스 파일 형태로 자율주행 세션이 누수 없이 안전하게 아카이빙된 것을 확인했습니다.

|**`/tf`**|tf2_msgs/msg/TFMessage|3,678개|로봇 각 부품 및 좌표계 간의 실시간 동적 변환 정보|

|**`/odom`**|nav_msgs/msg/Odometry|1,996개|바퀴 엔코더 기반의 누적 상대 주행 위치 정보|
|**`/local_plan`**|nav_msgs/msg/Path|1,204개|지역 제어기(Controller)가 계산한 실시간 장애물 회피 경로선|

|**`/amcl_pose`**|geometry_msgs/msg/PoseWithCovarianceStamped|60개|지도와 센서 데이터를 대조해 보정한 로봇의 추정 좌표[cite: 1]|
|**`/map`**|nav_msgs/msg/OccupancyGrid|1개|주행 당시 메모리에 올라와 있던 2D 점유 격자 지도 데이터[cite: 1]|

## 4. 실습 결론

- 쉘 스크립트를 통해 자율주행 자원 분석에 필요한 다수의 토픽을 누락 없이 일괄 수집하는 프로토콜을 검증했습니다[cite: 1].
    
- 특히 뼈대 정보인 좌표 변환(`/tf`, `/tf_static`)과 환경 정보인 지도(`/map`)가 함께 패키징되어 저장되었기 때문에, 물리 시뮬레이터(가제보)를 실행하지 않고도 RViz2 상에서 로봇의 주행 궤적과 주변 센서 반응을 완벽하게 오프라인 리플레이할 수 있는 유용한 데이터셋을 확보했습니다[cite: 1].
